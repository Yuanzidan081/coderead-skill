# CodeRead 任务切换器 (Task Switcher)

## 触发方式

- **手动触发**：通过 `/coderead-switch` 斜杠命令。
- **自动激活**：否——仅在用户明确需要切换活动任务时手动执行。

## 目标

让用户在多个代码阅读任务之间自由切换上下文。选择某个已有任务后，后续所有分析 Skill（`/coderead-birdview` `/coderead-dataflow` `/coderead-intent` `/coderead-verify`）的输出将定向到该任务文件夹，实现「回到某个已有的子文件夹继续学习」。

## 执行步骤

### 1. 扫描已有任务

使用 `list_directory` 工具列出 `coderead/` 目录下的所有子文件夹（排除 README.md 文件）。

### 2. 展示任务清单并读取状态

对每个任务文件夹：

1. 读取其 `README.md` 获取任务描述和当前进度。
2. 检查各报告文件是否存在，更新状态标记。

向用户呈现：

```markdown
---

## 📂 已有任务

| # | 任务文件夹 | 描述 | birdview | dataflow | intent | verify | notes | 最后更新 |
|---|----------|------|----------|----------|--------|--------|-------|---------|
| 1 | `auth-dataflow` | 认证模块数据流分析 | ✅ | ⬜ | ⬜ | ⬜ | ⬜ | 2026-06-15 |
| 2 | `order-event-sourcing` | 订单事件溯源研究 | ✅ | ✅ | ⬜ | ⬜ | 📝 | 2026-06-14 |
| 3 | `project-architecture` | 项目宏观架构梳理 | ✅ | ⬜ | ⬜ | ⬜ | ⬜ | 2026-06-10 |

> 当前活动任务：`auth-dataflow`

请输入任务编号（如 `2`）或文件夹名来切换。输入 `0` 取消。
---
```

### 3. 切换上下文

收到用户选择后：

1. 验证所选任务文件夹存在。
2. 将当前活动任务切换为该文件夹。
3. 显示该任务的 README 摘要和当前进度：

```markdown
---

## ✅ 已切换到任务：`order-event-sourcing`

**描述**：订单模块事件溯源设计意图研究
**进度**：2/4 报告已完成

| 文件 | 状态 |
|------|------|
| `birdview.md` | ✅ 已完成 |
| `dataflow.md` | ✅ 已完成 |
| `intent.md` | ⬜ 待生成 |
| `verify.md` | ⬜ 待生成 |
| `notes.md` | 📝 有记录 |

### 🔜 建议下一步

根据当前进度，推荐执行：
- `/coderead-intent` — 深挖事件溯源的设计权衡
- `/coderead-verify` — 验证你对事件溯源的理解

继续分析，所有输出将保存到 `coderead/order-event-sourcing/`。

---
```

### 4. 如果用户未指定

如果用户直接输入 `/coderead-switch` 而没有任何参数：
- 展示完整任务清单让用户选择。
- 如果 `coderead/` 下没有任何任务子文件夹，提示：`尚无任务。使用 /coderead-propose <你的问题> 创建第一个任务。`

**产出**：切换后的任务上下文 + 进度概览 + 下一步建议。

---

## 执行准则

1. **展示进度而非仅名字**：列出任务时必须附带各报告完成状态（✅/⬜），这样用户能快速判断哪个任务需要继续。
2. **智能推荐**：切换后根据未完成的报告自动推荐下一步执行的分析命令。
3. **若无任务则不报错**：`coderead/` 下没有任何任务子文件夹时，友好提示用户用 `/coderead-propose` 创建。
4. **仅管理上下文，不修改文件**：除了更新任务 README 中的文件状态外，不修改已有分析报告。
